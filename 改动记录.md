## Bug 修复（2026-07-11）

### Bug 1（致命）：Critic 在 rollout 时输入全为零

**位置**: train.py 原 193-199 行

**问题**: 构建 Critic 输入 `s_t` 时，`priv_batch[:, :137]` 对 73 维 priv 取不到 137 维，导致 `s_t.shape != state_dim`，回退到 `torch.zeros(256, 184)`。
**影响**: GAE 优势估计变成随机噪声，PPO 退化为 REINFORCE，学习效率极低。

**修复**: 新增 `_build_state()` 函数，将 obs + priv 拼接后补齐到 state_dim。所有 3 处调用点统一修复。

### Bug 2（严重）：privileged 维度不足

**位置**: dwl_env.py `_get_privileged_obs()`, dwl_config.py `privileged_dim`

**问题**: privileged_dim=73，论文要求 137。缺少的 64 维全为零，Critic 无法获取完整特权信息。
**修复**: privileged_dim → 137，重写 `_get_privileged_obs()` 包含：基座速度、角速度、重力投影、欧拉角、高度、DR参数(4项)、步态掩码、足部位姿/速度(12)、接触力(4)、力矩(12)、指令(3)、相位(2)、摆动状态(4)、关节状态(24)、历史动作(24)、episode进度(1)。

### Bug 3（次要）：`_build_state()` 三处调用未统一

**修复**: 引入单一辅助函数，buffer 存储、rollout value、GAE last_value 三处统一调用。
